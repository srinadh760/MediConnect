# MediConnect 🏥

Hey! This is my full-stack project, **MediConnect**. It's a comprehensive doctor appointment booking and medical consultation platform. I built it from scratch using React, Node.js, Express, and MySQL. 

I spent a lot of time trying to solve real-world backend problems like database concurrency and double-booking, and I'm pretty proud of how the architecture turned out!

## Features I Built 🚀

*   **Bitmask Scheduling Engine (My favorite part!):** Instead of storing each appointment as a separate row and doing slow, complex time-overlap queries, I implemented a 64-bit BigInt bitmask system in MySQL. Each 5-minute slot in a doctor's shift is represented by a single bit (0 for free, 1 for booked). When two users try to book the exact same slot at the exact same millisecond, the database uses atomic bitwise operations (`shift & mask`) to guarantee absolutely zero double-booking! It's blazing fast.
*   **AI Symptom Triage Chatbot:** I integrated the Google Gemini 2.5 Flash API. Patients can type in their symptoms, and the AI will analyze them and spit out search tags (like "cardiologist" or "skin") so patients know exactly which type of doctor to book. I even wrote a strict prompt so it never hallucinates fake tags and only searches our database! It also detects foreign languages and automatically filters for doctors who speak them.
*   **Virtual Wallet System:** Patients start with a virtual wallet balance. When they book, it deducts automatically, and if they cancel, it refunds instantly.
*   **Language & Specialization Filters:** Patients can filter doctors by what languages they speak and their specific medical tags. 
*   **Medical Records Management:** Both doctors and patients can upload and attach PDF/image medical records to specific appointments.

## Tech Stack 💻

*   **Frontend:** React 18, Vite, React Router, Lucide Icons. I completely overhauled the UI to a clean, professional "light medical" theme using mostly Vanilla CSS.
*   **Backend:** Node.js, Express.js.
*   **Database:** MySQL (v9.7). I used the `mysql2` pool to handle the BigInt strings properly to prevent JavaScript precision loss on the bitmasks.
*   **AI Integration:** `@google/generative-ai` SDK.

## How to Run It Locally 🛠️

If you want to spin this up on your machine to test it out, follow these steps:

### 1. Database Setup
Make sure you have MySQL running on port 3306. 
You'll need a database named `mediconnect`. 

### 2. Backend
Navigate to the backend folder:
```bash
cd backend
npm install
```
Create a `.env` file with your database credentials and Gemini API Key:
```env
PORT=5000
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=mediconnect
GEMINI_API_KEY=your_gemini_key
JWT_SECRET=some_secret_key
```
Then, generate the dummy data (this creates 100 fake doctors and patients):
```bash
node seed.js
```
Start the server:
```bash
npm run dev
```

### 3. Frontend
Open a new terminal and navigate to the frontend folder:
```bash
cd frontend
npm install
npm run dev
```

The website will be running on `http://localhost:5173`. 
You can use any of the fake emails generated by the seed script (password is always `password123`) or just click "Sign Up" to create a new patient account!
